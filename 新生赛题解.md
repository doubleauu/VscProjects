## B.假树人

知识点：模拟

本题记录一下当前最大值，每有一个树人苏醒，就让他变为最大值，若此时最大值不唯一，就再让这个树人高度再加1，变为唯一最大值，再更新最大值。

```c++
#include <bits/stdc++.h>
using i64 = long long;

void solve() {
    int n, m;
    std::cin >> n >> m;
    std::vector<int> v(n);
    for(int i = 0; i < n; ++i) {
        std::cin >> v[i];
    }
    int mx = *std::max_element(v.begin(), v.end());
    int cnt = std::count(v.begin(), v.end(), mx);
    for(int i = 0; i < m; ++i) {
        int id;
        std::cin >> id;
        id--;
        if(v[id] < mx || (v[id] == mx && cnt != 1)) {
            v[id] = mx + 1;
            mx = v[id];
            cnt = 1;
        }
    }
    std::cout << mx << '\n';
}

int main() {
    std::ios::sync_with_stdio(false);
    std::cin.tie(nullptr);
    int T = 1;
    std::cin >> T;
    while(T--) {
        solve();
    }
    return 0;
}
```

## C.源石搜寻

**知识点：排序，枚举**

我们首先需要确定，当有 $n$ 个坐标点出现在坐标轴上，其最小的矩形面积是多少？

显而易见，其最小的矩形面积只跟最小和最大的 $x$ 和 $y$ 数值有关，即其最小矩形面积为 $(max(x_i)-min(x_i)) * (max(y_i)-min(y_i))$ $(1 \le i \le n)$ 。

故选取的点，只需要分开关注两个集合最大和最小值即可。

可以发现，当点进行排序后，选择 $x$ 坐标时若我们选择了第 $i$ 个点，使其最小的选择便是第 $i+n-1$ 个点  ，那么 $y$ 选择剩下的点。

但是需要注意的是直接选择 $1-n$ 和 $n+1 - 2*n$两个集合是错误的，我们仅能知道上述前提条件下，其的最优解，故我们需要枚举所有长度为 $n$ 的子数组的所有选择，从中选择最小乘积。

枚举时由于数组已经有序，故使用两个集合端点直接进行计算即可。

整体时间复杂度集中在排序，故时间复杂度为 $O(nlogn)$。

```cpp
#include <bits/stdc++.h>
using namespace std;

using i64 = long long;
using i128 = __int128_t;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    int n;
    cin >> n;
    vector<i64> f(n << 1 | 1);
    for (int i = 1; i <= (n << 1); i++) {
        cin >> f[i];
    }

    sort(f.begin() + 1, f.end());

    i64 res = (f[n] - f[1]) * (f[n << 1] - f[n + 1]);
    for (int i = 2; i <= n; i++) {
        res = min(res, (f[n << 1] - f[1]) * (f[n + i - 1] - f[i]));
    }

    cout << res;
    return 0;
}
```

## E. 强风追踪

**知识点：前缀和，思维**

由于题目数据较大，暴力模拟显然无法通过。

##### 方法一 (逆向思维)

我们虽然无法记录所有香味并模拟其全部的移动，但是我们发现，中心点和人两个点是不变的，是容易模拟的。

我们便可以换一个思路，每次风吹时，不模拟所有香味沿着风向移动，而是中心和人一起移动，那么此时只需要判断人是否经过中心走过的路径即可。

由于坐标可能较大，空间不太允许直接开二维数组记录，所以需要用到$set、 map$等离散化存储经过路径的坐标点。

整体时间复杂度集中在存储使用的map，题解代码时间复杂度 $O(nlogn)$ 

```cpp
#include <bits/stdc++.h>
using namespace std;

using i64 = long long;
using u64 = unsigned long long;
using i128 = __int128_t;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    int n, r, c;
    string s;
    cin >> n >> r >> c >> s;

    map<pair<int, int>, bool> mp;
    mp[{0, 0}] = 1;
    int x = 0, y = 0;
    for (const auto& k : s) {
        if (k == 'D') {
            y++, c++;
        } else if (k == 'U') {
            y--, c--;
        } else if (k == 'L') {
            x++, r++;
        } else {
            x--, r--;
        }
        mp[{x, y}] = 1;

        cout << mp.count({r, c});
    }
    return 0;
}
```

##### 方法二 (前缀和做法)

由于每次结束之后都会在中心位置产生新的香味，那么判断人位置是否有香味其实就是是否存在一个香味从产生某个开始$t=L$到当前$t=R$能否移动到人的位置，那么其移动可以用求和$[L,R]$期间的移动即可得到答案。

类似于求一个数组内，有多少子数组的和等于固定数值 $k$ 。

那么此时通过存储前缀和来优化询问即可得到答案。

那么到达第$t$秒时，此时前缀和数组到达$prex_t$与$prey_t$ ,那么只需要查询前面是否存在一个为$prex_t - r$与$prey_t-c$的前缀即可，这个也可以通过 $map、set$ 等存储和查找。

整体时间复杂度集中在存储需要的map,时间复杂度为 $O(nlogn)$ 。

```cpp
#include <bits/stdc++.h>
using namespace std;

using i64 = long long;
using u64 = unsigned long long;
using i128 = __int128_t;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    int n, r, c;
    string s;
    cin >> n >> r >> c >> s;

    map<pair<int, int>, bool> mp;
    int x = 0, y = 0;
    mp[{0, 0}] = 1;
    for (const auto& k : s) {
        if (k == 'R') {
            x++;
        } else if (k == 'U') {
            y++;
        } else if (k == 'L') {
            x--;
        } else {
            y--;
        }
        
		mp[{x, y}] = 1;
        cout << mp.count({x - r, y - c});
    }
    return 0;
}
```

## F. 天酱的 min * max

**知识点：数学，递推**

 对于这个问题，暴力地枚举每个子序列的时间复杂度是 $O($$2^n$) ，这个做法显然会超时，我们需要考虑对其优化。

由于对于每个子序列的贡献只与其最大值与最小值有关，因此我们无需关心其中间值，只需要获取其最大值与最小值。对此我们可以先将数组排序后再处理，这样便于我们获取其最值。设排序后的数组为：$ {a_1,a_2,......,a_n} $

因为只与最大值与最小值有关，考虑枚举最大值与最小值。对于计算最小值为 $a_i$  ，最大值为 $a_j$ 的子序列个数。当 $i=j$ 时，显然此时只有一个子序列满足要求，即为 ${a_i}$ 。当 $i \ne j$ 时，$i$ 和 $j$ 之间有 $(j - i - 1)$ 个数，每个数字出现与否相互独立，共有 $(2^{j-i-1}) $个子序列满足要求，因此其对答案的贡献是 $a_i * a_j * 2^{j-i-1}$ ，对应的代码如下：

```cpp
#include <bits/stdc++.h>
using namespace std;
using i64 = long long;
const int MOD = 1e9 + 7;
const int N = 1e6;
void solve(){
    int n;
    cin >> n;
    vector<int>a(n + 1);
    for (int i = 1;i <= n;i++){
        cin >> a[i];
    }
    sort(a.begin() + 1, a.end());
    i64 res = 0;
    for(int i = 1;i<=n;i++){//以a[i]为最小值
    	res = (res + 1ll * a[i] * a[i])%MOD;//i==j时的处理,1ll是1,不是111!,用来将a[i]*a[i]转换为long long防止溢出
    	for(int j = i+1;j<=n;j++){//以a[j]为最大值
    		res = (res + 1ll * a[i] * a[j] % MOD * num[j - i - 1] % MOD)%MOD ;
    	}
    }
    cout << res << '\n';
}
signed main() {
	num[0] = 1 ;
	for(int i = 1;i<N;i++){//预处理2的次方的值,就无需使用快速幂进行计算了
		num[i] = num[i-1] * 2 % MOD ;
	}    
	ios::sync_with_stdio(0);
    cin.tie(0);//这两行是关闭同步流,不加也行
    int orz = 1;
    cin >> orz;
    while (orz--) {
        solve();
    }
}
```

此时时间复杂度到了$O(n^2)$级别，仍然无法不能通过此题，需要继续优化。

**(通俗的讲)**实际上就是对于同一个最小值，$a_{j}$ 到 $a_{j+1}$ 时多了一个数( $a_j$ )，如最小值都为 $a_1$ 时，$a_{j+1}$ 与 $a_1$ 之间数字个数比 $a_j$ 到 $a_i$ 多 $1$ 个数( $a_j$ )，因此要 $*2$，还要加上子序列只有自己本身一个数的情况，得出的值就为以$a_{j+1}$为最大值的所有子序列的贡献值。

**(较为严谨的证明)**观察到对于以 $a_j$ 为最大值的所有子序列的贡献为:
$$
a_j*a_j+\sum_{i=1}^{j-1}(a_j*a_i*2^{j-i-1}) = a_j*(\sum_{i=1}^{j-1}(a_i*2^{j-i-1}))+a_j*a_j
$$
对于每个的$a_j*a_j$我们可以单独计算，只需要考虑如何快速计算$ a_j*(\sum_{i=1}^{j-1}(a_i*2^{j-i-1}))$

令 $t = a_j*(\sum_{i=1}^{j-1}(a_i*2^{j-i-1}))$

对于以 $a_{j+1}$ 为最大值的所有子序列的贡献为: $a_{j+1}*(\sum_{i=1}^{j}(a_i*2^{j-i}))+a_{j+1}*a_{j+1}$ 

将 $(\sum_{i=1}^{j}(a_i*2^{j-i}))与(\sum_{i=1}^{j-1}(a_i*2^{j-i-1}))$ 做差，

结果为: $a_j*(\sum_{i=1}^{j-1}(a_i*2^{j-i-1})+a_j$ ，即为 $t+a_{j}$ ,

因此 $a_{j+1}*(\sum_{i=1}^{j}(a_i*2^{j-i})+a_{j+1})$ 即为 $a_{j+1}*((t*2+a_{j})+a_{j+1})$ ,

因此，我们可以用 $t$ 来快速计算最大值为 $a_j$ 到 $a_{j+1}$ 时要乘的值。**需要注意还要加上当子序列为 $\{a_{j+1}\}$ 时的 $a_{j+1}*a_{j+1}$**  

```cpp
#include <bits/stdc++.h>
using namespace std;
using i64 = long long;
const int MOD = 1e9 + 7;
void solve(){
    int n;
    cin >> n;
    vector<int>a(n + 1);
    for (int i = 1;i <= n;i++){
        cin >> a[i];
    }
    sort(a.begin() + 1, a.end());
    i64 res = 0;
    i64 t = 0;
    for (int i = 1;i <= n;i++){
        res = (res + 1ll * t * a[i] % MOD + 1ll * a[i] * a[i] % MOD) % MOD;
        t = (t * 2 + a[i]) % MOD;
    }
    cout << res << '\n';
}
signed main() {
    ios::sync_with_stdio(0);
    cin.tie(0);
    int orz = 1;
    cin >> orz;
    while (orz--) {
        solve();
    }
}
```

## G. k 进制？

知识点：模拟

1. 首先需要确定哪些字符是 k 进制合法的：0-9，A-Z（大写）对应 10-35。
2. 遍历字符串，找出所有连续的合法字符片段。
3. 比较这些片段，找出最大的那个（注意前导零的处理）。

```c++
#include<bits/stdc++.h>
using namespace std;
int main(){
    int n,k;
    cin >> n >> k;
    string s;
    cin >> s;
    s = ' ' + s;
    vector<string>ans;
    string now="";
    if(k <= 10){
       for(int i = 1;i <= n;i++){
           if(s[i] <= '0' + k - 1 && s[i] >= '0'){
              now += s[i];
           }
           else{
              ans.push_back(now);
              now = "";
           }
       }
    }
    else{
       for(int i = 1;i <= n;i++){
           if(s[i] <= '9' && s[i] >= '0'){
              now += s[i];
           }
           else if(s[i] <='A' + (k - 11) && s[i] >= 'A'){
              now += s[i];
           }
           else{
            //   cout << now;
              ans.push_back(now);
              now = "";
           }
       }
    }
    if(now.size() != 0){
        ans.push_back(now);
    }

    for(string i:ans){
        int flag = 0;
        now = "";
        for(int j = 0;j < i.size();j++){
            if(i[j] != '0'){
                flag = 1;
            }
            if(flag == 1){
                now += i[j];
            }
        }
        if(now.size() >= 1){
            tans.push_back(now);
        }
        else{
            tans.push_back("0");
        }
    }
    if(tans.size() == 0){
        cout << -1 ;
    }
    else{
        now = tans[0];
        for(string i:tans){
            if(i.size() > now.size()){
                now = i;
            }
            else if(i.size() == now.size()){
```

## H. 智慧 or 力量

知识点：优先队列

1. 这是一个类似 BFS 的问题，但需要总是选择当前能击败的敌人中力量最小的。
2. 使用优先队列（最小堆）来实现：总是先处理力量值最小的可达敌人。
3. 将所有可达敌人入队列，每次弹出力量最小的敌人，如果这个敌人都大于当前智慧值证明没有能击败的敌人了，否则可以击败，智慧值更新为当前智慧值加击败的敌人的力量值。

```c++
#include<bits/stdc++.h>
using namespace std;
#define int long long
void solve(){
    int n,m;
    cin >> n >> m;
    int x,y,M;
    cin >> x >> y >> M;
    vector<vector<int>>v(n + 1,vector<int>(m + 1));
    for(int i = 1;i <= n;i++){
        for(int j = 1;j <= m;j++){
            cin >> v[i][j];
        }
    }
    priority_queue<pair<int,pair<int,int>>, vector<pair<int, pair<int,int>>>,greater<pair<int, pair<int,int>>>>q;
    int cnt = 0;
    q.push({v[x][y],{x,y}});
    int dx[4] = {-1, 0, 1, 0};
    int dy[4] = {0, 1, 0, -1};
    while(!q.empty()){
        auto a = q.top();
        int cx = a.second.first;
        int cy = a.second.second;
        if(a.first > M){
            break;
        }
        M += a.first;
        q.pop();
        for(int i = 0; i < 4; i++){
            int nx = cx + dx[i];
            int ny = cy + dy[i]; 
                  
            if(nx >= 1 && nx <= n && ny >= 1 && ny <= m && v[nx][ny] != 0){
                q.push({v[nx][ny], {nx, ny}});
                v[nx][ny] = 0;
            }
       }
       cnt++;
       
    }
    cout << cnt - 1 << "\n";
}

signed main(){
    int _ = 1;
    cin >> _;
    while(_--){
        solve();
    }
}
```

## I. na 是最小值还是最大值

知识点：单调栈

1. 可以使用单调栈求解每个元素作为最小值的最大区间长度。
2. 对于每个元素 $a_i$，找到它能成为最小值的最大区间长度 L。
3. 然后对于每个区间长度，找到他可能的最大元素。

```c++
#include <bits/stdc++.h>
using i64 = long long;

void solve() {
    int n;
    std::cin >> n;
    std::vector<int> v(n + 1);
    for(int i = 1; i <= n; ++i) {
        std::cin >> v[i];
    }
    std::vector<int> l(n + 1), r(n + 1);
    std::stack<int> stk;
    for(int i = 1; i <= n; ++i) {
        while(!stk.empty() && v[stk.top()] >= v[i]) {
            stk.pop();
        }
        l[i] = stk.empty() ? 0 : stk.top();
        stk.push(i);
    }
    while(!stk.empty()) stk.pop();
    for(int i = n; i >= 1; --i) {
        while(!stk.empty() && v[stk.top()] >= v[i]) {
            stk.pop();
        }
        r[i] = stk.empty() ? n + 1 : stk.top();
        stk.push(i);
    }
    std::vector<int> ans(n + 1);
    for(int i = 1; i <= n; ++i) {
        ans[r[i] - l[i] - 1] = std::max(ans[r[i] - l[i] - 1], v[i]);
    }
    for(int i = n - 1; i >= 1; --i) {
        ans[i] = std::max(ans[i], ans[i + 1]);
    }
    for(int i = 1; i <= n; ++i) {
        std::cout << ans[i] << " \n"[i == n];
    }
}

int main() {
    std::ios::sync_with_stdio(false);
    std::cin.tie(nullptr);
    int T = 1;
    std::cin >> T;
    while(T--) {
        solve();
    }
    return 0;
}

```

## K. MEX

**知识点：二分**

注意到，题目所求的最小值具有二分性，即若 $x$ 可以满足，更小的答案肯定也能满足，其满足单调性。

显而易见，我们可以二分答案，每次暴力模拟，若当前已经满足要求了，便直接在此分组，最终判断是否满足 $k$ 组的要求即可。

整体时间复杂度: $O(nlogn)$ 。

```cpp
#include<bits/stdc++.h>
using namespace std;

using i64 = long long ;
using i128 = __int128_t;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    int n, k;
    cin >> n >> k;
    vector<int> f(n + 1);
    for (int i = 1; i <= n; i++) {
        cin >> f[i];
    }

    auto check = [&](int x) -> bool {
        vector<int> vis(x);
        int cnt = 0, sum = 0;
        for (int i = 1; i <= n; i++) {
            if (f[i] < x) {
                sum += (vis[f[i]] == 0);
                vis[f[i]] = 1;
            }

            if (sum >= x) {
                vis.assign(x, 0);
                cnt++;
                sum = 0;
            }
        }
        return cnt >= k;
    };

    int l = 0, r = n + 1;
    while (l <= r) {
        int mid = (l + r) >> 1;
        if (check(mid)) {
            l = mid + 1;
        } else {
            r = mid - 1;
        }
    }

    cout << r << "\n";
    return 0;
}
```

## L. 凉宫春日问题

本题要求判断给定的序列 $S$ 是否为 $n$ 的"超排列"。根据题目定义，超排列需要满足两个条件：

1.  包含 $\{1, 2, \dots, n\}$ 的**所有全排列**作为连续子序列。
2.  每个全排列在 $S$ 中**恰好出现一次**。

数据范围给出 $n \le 8$，这意味着全排列的总数 $n!$ 最多为 $8! = 40320$，这是一个相对较小的数字。序列长度 $L$ 最大为 $10^5$。


由于 $n$ 较小，我们可以直接生成所有 $n!$ 个全排列，然后遍历序列 $S$ 进行统计。

### 步骤如下：

1.  **生成全排列集合**：
    使用 `next_permutation` 生成 $1$ 到 $n$ 的所有全排列，并将它们存储在一个集合（如 `set` 或哈希表）中，以便快速查询。同时，我们需要知道全排列的总数。

2.  **遍历序列子串**：
    遍历输入序列 $S$，检查每一个长度为 $n$ 的连续子序列（窗口）。
    对于序列 $S = (W_1, W_2, \dots, W_L)$，我们需要检查从下标 $i$ 开始的子串 $(W_i, W_{i+1}, \dots, W_{i+n-1})$，其中 $0 \le i \le L-n$。

3.  **统计出现次数**：
    使用一个映射（`map` ）来记录每个全排列在 $S$ 中出现的次数。
    - 如果当前子串是一个合法的全排列（即存在于步骤1生成的集合中），则将其计数加1。
    - 如果当前子串不是全排列（包含重复元素），则忽略。

4.  **验证条件**：
    遍历统计结果，检查是否满足超排列的定义：
    1.统计到的不同全排列数量必须等于 $n!$
    2.每个全排列的出现次数必须为1。

5.  **时间复杂度分析**：
    -   生成全排列：$O(n \cdot n!)$。
    -   遍历序列：序列长度为 $L$，每个窗口长度为 $n$，比较或哈希的时间为 $O(n)$,总共 $O(n \cdot L)$。
    -   总时间复杂度：$O(n \cdot (n! + L))$。
    -   代入 $n=8, L=10^5$，计算量约为 $8 \times (40320 + 100000) \approx 1.1 \times 10^6$，远低于通常的 $10^8$ 限制，可以轻松通过。

## 代码实现 (C++)

本体代码并不难，只要熟悉STD库的使用便可轻松实现，以下是完整代码示例：


## M. 罗德岛源石污染区作战

**知识点：数学，前缀和，枚举, 贪心**

由于线段之间可以重叠，贪心思想容易想到如果选择了$[l_i,r_i]$和$[l_j,r_j]$两个线段后，选择标记线段一定是$[l_i,r_j]$才能取得最大报酬。

我们将每个标记线段中选择左端点的和右端点的分成两个集合分别为 $L$ 和 $R$ 。

先定义有偶数个线段时的答案，容易得到答案的表达式为$\sum_{i \in R}r_i-\sum_{j \in L}l_i$ 

即:
$$
\sum_{i \in R}r_i-\sum_{j \in L}l_j=\sum_{i \in R}r_i-\sum_{j \in L}l_j+\sum_{j \in L}r_j-\sum_{j \in L}r_j
$$
定义 $U$ 为所有选择标记的线段的全集

得到原式等于
$$
\sum_{i \in U}r_i - \sum_{j \in L}(l_j+r_j)
$$
可以发现$\sum_{i \in U}r_i$是一个定值，那我们仅需要最小化$\sum_{j \in L}(l_j+r_j)$即可，仅需要存下所有线段的左右端点坐标之和，进行排序即可得到。

那么奇数情况就很容易得到了，我们只需要枚举哪一个是剩下的线段，求其长度然后照搬偶数做法即可。

但是肯定不能重新开始，我们需要预处理上述答案，对排序后的结果进行前缀和，那么每次枚举时我们仅需要$O(1)$的时间即可得到答案，那么只需要枚举剩余线段，通过前缀和和后缀和计算得到，复杂度为 $O(n)$ 。

**注意：不要忘记加上全部线段的基础报酬哦~**

主要时间复杂度都集中在排序，故整体时间复杂度为$O(nlogn)$。

```cpp
#include <bits/stdc++.h>
using namespace std;

using i64 = long long;
using i128 = __int128_t;

void solve() {
    int n;
    cin >> n;
    vector<pair<i64, i64>> f(n + 1);
    i64 sum = 0;
    for (int i = 1; i <= n; i++) {
        cin >> f[i].first >> f[i].second;
        sum += f[i].second;
        sum += f[i].second - f[i].first;
    }

    sort(f.begin() + 1, f.end(), [&](auto x, auto y) { return x.first + x.second < y.first + y.second; });

    vector<i64> pre(n + 1);
    for (int i = 1; i <= n; i++) {
        pre[i] = pre[i - 1] + f[i].first + f[i].second;
    }

    i64 ans = 0;
    if (n & 1) {
        for (int i = 1; i <= n; i++) {
            if (i <= n / 2) {
                ans = max(ans, sum - pre[(n + 1) / 2] + f[i].first);
            } else {
                ans = max(ans, sum - pre[n / 2] - f[i].second);
            }
        }
    } else {
        ans = sum - pre[n / 2];
    }

    cout << ans << "\n";
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    int t;
    cin >> t;
    while (t--) {
        solve();
    }
    return 0;
}
```

## N. 无穷完美二叉搜索树

最近公共祖先板子题

题目上的条件其实是求节点 $a, b$ 的最近公共祖先，由于树高不超过 $60$ ，只要能知道节点 $x$ 的父亲和在第几层即可暴力求出来。根据观察可以知道以叶子节点为第 0 层，第 $x$ 层的节点 $g$ 都为 $2 ^ x$，也可以比较容易看出来节点 $x$ 的父亲为节点 $(x - g_x) | (g_x * 2)$ 。然后暴力跳父亲即可找到最近祖先。

时间复杂度 $ O(Tlogn)$。

```c++
#include <bits/stdc++.h>
using namespace std;
using u32 = uint32_t;
using i64 = int64_t;
using u64 = uint64_t;
using f64 = long double;
using i128 = __int128_t;
using u128 = __uint128_t;
i64 G(i64 x) {
    return x & -x;
}
void solve() {
    i64 x, y;
    cin >> x >> y;
    while (x != y) {
        i64 gx = G(x);
        i64 gy = G(y);
        if (gx <= gy) {
            x -= gx;
            x |= gx << 1;
        } else {
            y -= gy;
            y |= gy << 1;
        }
    }
    cout << x << "\n";
    return;
}
int main() {
    ios::sync_with_stdio(false);
    cin.tie(0), cout.tie(0);
    int T = 1;
    cin >> T;
    while (T--) {
        solve();
    }
    return 0;
}
```
